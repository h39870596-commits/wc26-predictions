# WC26 AI 数据分析 · 2026 美加墨世界杯

**AI 大数据体育数据分析平台**(中英双语):基于「五维分析法」与多智能体联网调研的世界杯赛事数据分析站。定位为数据分析与研究参考,**非博彩网站**——不提供任何投注功能,公开字段禁用盘口/赔率措辞,全部结论带来源可溯。

纯静态站点,无后端依赖:`index.html`(渲染层)+ `assets/data.js`(数据层)。

## 目录结构

```

assets/data.js          数据层 —— 页面唯一数据来源(由脚本生成,勿手改)
scripts/build_data.py   构建脚本:data/*.json → assets/data.js(含去博彩化兜底清洗)
data/wc.json            48 队基础信息 + 72 场小组赛赛程(actual = 真实比分)
data/predictions.json   全程推演结果(五维评分/小组/淘汰赛/冠军)
data/matchday-*.json    每个比赛日的深度调研档案(含来源;每场附 "en" 英文块;
                        可选 "featured_no" 指定精选场次,缺省取置信度最高一场)
data/review.json        预测 × 实际复盘记录(中文字段 + label_en/note_en/title_en/summary_en)
data/recaps.json        已赛场次复盘档案(双语叙事/时间线/数据/来源,逐日追加)
data/history.json       世界杯历史数据(1930–2022 冠军谱系/夺冠榜/射手榜/纪录,带来源)
data/experts.json       专家与模型观点(双语,带原始报道链接;赛程中可定期刷新)
data/site-data.json     球队展示数据(中文名/旗帜/五维/综合分)
```

## 付费模型(当前为演示模式)

- **单日精选 ¥9.9**:解锁当日精选场次的 AI 预测报告(精确比分/概率/置信度/深度分析),仅当日有效(localStorage 存日期,跨日自动失效)。
- **全程订阅 ¥98**:赛事期间每日精选报告(直至决赛)+「全程推演」沙盘页。
- 关键约束:**预测按日推送,每日仅 1 场精选**,不一次性发布未来赛果;非精选场次的精确预测不发售,赛后在复盘中公开(透明可核验)。
- 构建层面:`build_data.py` 只把精选场次的 `paid` 块写进 `data.js`,其余场次的精确预测不出库。

### 接真实支付(用户已注册微信/支付宝商家收款码)

⚠️ **静态商家收款码只能收款,无法自动确认订单、自动发货解锁**,两条路线:

1. **推荐:商户 API 收单**(需后端,可用 Serverless/云函数):微信支付 Native/JSAPI + 支付宝当面付/手机网站支付 → 后端创建订单 → 回调验签 → 发放解锁 token;前端把 `WCPay.checkout(channel, product)` 替换为「创建订单 → 拉起收银台 → 凭 token 解锁」。
2. **过渡:收款码 + 人工/半自动发码**:页面展示收款码 + 订单号,用户支付后凭单号兑换解锁码;体验差、有对账成本,只适合极小流量验证期。

⚠️ **演示版的付费内容仍打包在前端 `data.js`(精选场次的 paid 块),查看源码可绕过,不可直接商用**。正式上线必须把付费数据移到服务端凭 token 下发。

## 每日自动化更新流程

比赛日结束后执行(可交给 Claude Code 定时任务):

1. **回填赛果**:真实比分写入 `data/wc.json` 对应场次 `actual`;
2. **复盘**:`data/review.json` 填 `actual`、`hit`(full/dir/miss)、`note`+`note_en`,`state` 改 `done`;
   待完赛(`pending`)日的 `note` 不要写概率/置信度等付费信息——渲染层会在未完赛时连同比分一起密封,完赛后才展示;
   并联网调研完赛场次,把双语复盘档案(叙事/时间线/转折点/数据/来源)追加进 `data/recaps.json`;
3. **新比赛日**:生成 `data/matchday-YYYY-MM-DD.json`(深度调研:天气/身价/伤病/市场预期,带来源+事实核查),
   每场附 `en` 英文块(headline/lean/analysis/factors/env);在 `data/review.json` 追加新的一天;
   **行文规范:公开字段禁用盘口、赔率、博彩公司名与美式赔率数字,市场信息一律写「隐含胜率 X%」「市场预期」**
   (build 脚本另有正则兜底清洗);
4. **构建并发布**:

```bash
python3 scripts/build_data.py
git add -A && git commit -m "update: matchday YYYY-MM-DD" && git push   # GitHub Pages 自动发布
```
